Barangay eBlotter System Flowchart

Table 3: Flow Chart

INPUT

  • Identification of problems in manual blotter recording.
  • Data collection from barangay staff and residents via interviews and surveys.
  • Analysis of data to determine system requirements and desired features.
  • Existing Barangay Ordinances & Legal Frameworks.

PROCESS

  • Design system workflow and interface (UI/UX + ERD).
  • Agile Iterative Development Model (Plan → Design → Develop → Test → Evaluate).
  • Implementation and Testing: unit testing, integration testing, alpha testing, and acceptance testing.
  • Deployment to local server or cloud technologies.

OUTPUT

  • Barangay eBlotter System
  • A Web-Based Digital Platform to Improve Incident and Complaint Reporting.
  • Enhance Transparency and Improve Administrative Processes in the Barangay.
  • User Training & Feedback Integration.
